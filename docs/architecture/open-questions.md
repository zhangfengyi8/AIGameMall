## 当前建议默认决策

如果没有额外要求，建议第一版采用以下默认方案：

- 只实现本地开发，不做部署。
- 使用 FastAPI。
- API 服务直接调用 Agent 编排模块，不拆独立 HTTP 服务。
- 不使用数据库，只读取 JSON。
- 第一批样例账号准备 20 条。
- 第一版只支持王者荣耀账号，同时字段预留多游戏。
- AI 导购接口支持流式响应，建议使用 SSE。
- 没有模型 API Key 时使用规则兜底。
- 推荐卡片点击后复用现有详情弹窗。
