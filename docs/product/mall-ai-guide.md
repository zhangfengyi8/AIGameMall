# 产品功能说明：商城页与 AI 导购

## 页面目标

当前第一版产品是一个游戏账号商城页面，用户可以直接浏览账号商品，也可以通过右下角 AI 悬浮球打开导购对话窗口，用自然语言描述需求，由 AI 推荐匹配账号卡片。

## 核心用户路径

1. 用户进入商城首页。
2. 页面展示游戏账号商品列表、搜索框、筛选项和基础商品信息。
3. 用户可以手动搜索或筛选账号。
4. 用户点击右下角 AI 悬浮球。
5. 页面打开 AI 对话窗口。
6. 用户输入需求，例如“预算 500，想要王者荣耀全英雄，皮肤多一点，风险低”。
7. 后端理解用户需求并查询 JSON 账号数据。
8. AI 返回推荐账号卡片、推荐理由、性价比、公允估价和风险提示。
9. 用户点击推荐卡片查看账号详情。

## 当前前端形态

前端入口：`frontend/index.html`

已具备的页面元素：

- 商城 Header。
- Banner 区域。
- 搜索区域。
- 热门搜索标签。
- 商品分类 Tab。
- 左侧筛选栏。
- 账号商品卡片列表。
- 账号详情弹窗。
- 右下角 AI 悬浮球。
- AI 对话窗口。
- AI 推荐账号卡片样式。

## 第一版范围

第一版优先完成从静态模拟数据到后端真实接口的替换。当前第一版只支持王者荣耀账号，样例账号准备 20 条。

包含：

- 从 JSON 文件读取 20 条王者荣耀账号样例数据。
- 商城列表从 API 获取数据。
- AI 对话请求调用 FastAPI 后端接口。
- AI 通过流式响应逐步返回导购回复和推荐账号卡片。
- 前端渲染推荐理由、估价和风险提示。

暂不包含：

- 用户登录。
- 支付交易。
- 卖家发布。
- 管理后台。
- 数据库。
- 生产部署。
- 复杂客服系统。
- 真实语音识别。

## 页面模块与后端关系

| 页面模块 | 后端能力 |
| --- | --- |
| 商城商品列表 | 账号列表查询 API |
| 搜索框 | 账号关键字查询 API |
| 筛选栏 | 账号条件筛选 API |
| 详情弹窗 | 账号详情 API |
| AI 对话窗口 | AI 导购对话 API |
| AI 推荐卡片 | 推荐结果响应结构 |

## 推荐卡片应展示的信息

- 账号 ID。
- 标题。
- 游戏名称。
- 当前段位。
- 价格。
- 系统估价。
- 性价比标签。
- 匹配度。
- 核心皮肤/英雄标签。
- 换绑状态。
- 风险等级。
- 风险提示。
- 推荐理由。

## 第一版验收标准

- 商城页面能从后端加载账号列表。
- 用户能打开 AI 对话窗口。
- 用户输入自然语言需求后能收到后端响应。
- 后端能基于 JSON 数据返回推荐账号卡片。
- 推荐结果包含推荐理由、估价和风险提示。
- 用户点击推荐卡片能查看对应账号详情。
